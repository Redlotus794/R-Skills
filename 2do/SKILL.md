# 2do 待办事项管理

## 目录结构

```
todo/
├── backlog/      # 待规划任务
├── todo/         # 进行中任务
├── completed/    # 已完成任务
├── archive/      # 历史简报归档
└── report_*.md   # 最新任务简报

references/
├── task_template.md          # 任务模板
├── report_template.md        # 简报模板
└── completion_template.md    # 完成总结模板
```

## 使用方式

### 创建任务
1. 在 `backlog/` 中创建新任务文件
2. 文件命名格式：`编号_分类_任务名称.md`（例如：`001_SAP_学习SAPBTP.md`）
   - 编号：三位数字全局递增（001, 002, 003...）
   - 分类：任务所属类别（如 SAP、学习、工作等）
   - 任务名称：简洁描述任务内容
3. 使用 Markdown 格式编写任务详情
4. 截止日期：如果用户未指定，询问用户或设为"暂未设定"

### 任务流转
- **backlog → todo**: 任务开始执行时移动
- **todo → completed**: 任务完成时移动
- **todo → backlog**: 任务暂停时移动

### 任务模板
```markdown
# 任务标题

## 优先级
- [ ] 高
- [ ] 中
- [ ] 低

## 状态
- [ ] 待处理
- [ ] 进行中
- [ ] 已完成

## 创建日期
YYYY-MM-DD

## 截止日期
YYYY-MM-DD 或 暂未设定

## 备注
任务详细描述

## 子任务
- [ ] 子任务1
- [ ] 子任务2
```

## 操作命令

### 创建新任务
```
在 backlog/ 目录创建任务文件
```

### 移动任务
```
将任务文件在不同目录间移动
```

### 查看任务
```
按目录查看不同状态的任务
```

### 生成任务简报
```
在 todo/ 目录下创建简报文件，汇总任务数量和进行中的任务内容
简报文件名：report_YYYY-MM-DD.md
历史简报移动到 archive/ 目录下
```

## 注意事项
- 所有任务文件会自动同步到百度网盘
- 定期清理 completed 目录中的旧任务
- 使用清晰的任务名称便于搜索

## 模板使用

### 任务模板
使用 `references/task_template.md` 创建新任务，包含优先级、状态、创建日期、截止日期、备注和子任务等标准字段。

### 简报模板
使用 `references/report_template.md` 生成任务简报，包含任务统计、进行中的任务、本周重点和下周计划等内容。

### 完成总结模板
使用 `references/completion_template.md` 记录任务完成情况，包含任务概述、主要成果、遇到的问题、经验总结和后续建议等内容。
