# VibeCoding 报告生成

## 概述

读取 `data/` 目录下的评测样例文件，提取每个案例的评分信息，生成一份汇总评分报告。
输出应聚焦统计结果与结论，不是单个案例的复写。

## 工作流

1. 扫描 `data/` 下各类型目录（`codegen`、`bugfix`、`test`、`diff`）中的案例 Markdown 文件。
2. 从每个案例提取基础字段：案例名称、类型、审核时间、总分（/100）。
3. 按类型聚合统计：案例数、平均分、最高分、最低分。
4. 计算全量汇总：总案例数、总体平均分、类型排名。
5. 提炼主要结论：高分模式、低分风险、数据缺口（缺失总分或字段不完整）。
6. 生成一份中文 Markdown 汇总文件。

## 数据范围

默认只使用 `data/` 中已有文件，不从 `template/` 推导评分结果。

类型与目录映射：
- `codegen`：`data/codegen/`
- `bugfix`：`data/bugfix/`
- `test`：`data/test/`
- `diff`：`data/diff/`

文件命名优先遵循 `data/README.md` 中规范：
`[谁]_[日期]_[类型]_[需求编号/业务编号].md`

## 评分规则

优先使用案例文件中已给出的 `总分（/100）`。

若存在明细表但缺失总分，可按权重补算；否则标记为“未计分”，并在汇总中单列。

## 证据与异常处理

必须列出数据来源文件路径。
当出现以下情况时，写入“数据异常”章节：
- 文件无法识别类型
- 缺失总分或关键字段
- 总分超出 `0-100`

## 输出约束

输出一份中文 Markdown 汇总评分文件，建议命名：
`data/summary/vibecoding-score-summary-YYYYMMDD.md`
优先参考 `references/汇总报表模板.md` 的章节与表格结构输出。

除非用户指定格式，默认章节顺序：
1. 汇总说明（统计时间、数据范围）
2. 各类型评分统计表（案例数、均分、最高、最低）
3. 全量总体统计（总案例数、总体均分、类型排名）
4. 关键结论（优势、风险、改进建议）
5. 数据异常与缺口（如有）
6. 数据来源清单

不要在报告正文之外输出分析草稿或内部推理过程。
